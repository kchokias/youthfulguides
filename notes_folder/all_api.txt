Youthful Guides - Private Backend API Documentation

This document describes all available private API endpoints for the Youthful Guides project backend.

⸻

User APIs:
	1.	Login
Method: POST
URL: /api/User/Login
Authentication: No
Body Example:
{ “email”: “user@example.com”, “password”: “123456” }
Response Example:
{ “success”: true, “token”: “JWT_TOKEN”, “user”: { “id”: 1, “username”: “johndoe”, “role”: “traveler” } }

⸻

	2.	Create New User
Method: POST
URL: /api/User/CreateNewUser
Authentication: No
Body Example:
{ “name”: “John”, “surname”: “Doe”, “username”: “johndoe”, “email”: “john@example.com”, “password”: “123456”, “role”: “traveler”, “region”: “Crete”, “country”: “Greece” }
Response Example:
{ “success”: true, “message”: “User created successfully”, “userId”: 23 }

⸻

	3.	Get User ID From Token
Method: GET
URL: /api/User/GetUserIdFromToken
Authentication: Yes (JWT)
Response Example:
{ “success”: true, “userId”: 23 }

⸻

	4.	Get User By User ID
Method: GET
URL: /api/User/GetUserByUserId/:id
Authentication: Yes (JWT)
Response Example:
{ “success”: true, “data”: { “id”: 23, “name”: “John”, “surname”: “Doe”, “username”: “johndoe”, “email”: “john@example.com”, “role”: “traveler”, “region”: “Crete”, “country”: “Greece”, “created_at”: “2025-04-01T10:00:00Z” } }

⸻

Guide APIs:
	5.	Get Guide Profile
Method: GET
URL: /api/GuideProfile/:id
Authentication: No
Response Example:
{ “guide_id”: 23, “username”: “guidename”, “name”: “George”, “surname”: “Smith”, “country”: “Greece”, “region”: “Crete”, “description”: “Experienced mountain guide.”, “profile_picture”: “data:image/jpeg;base64,…”, “average_rating”: 4.5, “total_bookings”: 5, “media”: [] }

⸻

	6.	Get Guide Reviews
Method: GET
URL: /api/GuideReviews/:guideId
Authentication: No
Response Example:
{ “guide_id”: 23, “reviews”: [ { “user_id”: 5, “username”: “traveler1”, “rate”: 5, “comment”: “Amazing experience!”, “reviewed_at”: “2025-04-10T12:00:00Z” } ] }

⸻

	7.	Upload Guide Media
Method: POST
URL: /api/Guide/UploadMedia
Authentication: Yes (JWT)
Body Example:
{ “guideId”: 23, “mediaData”: [“data:image/jpeg;base64,…”, “data:image/jpeg;base64,…”] }
Response Example:
{ “success”: true, “message”: “Media uploaded successfully”, “uploadedCount”: 2 }

⸻

	8.	Get All Guide Media
Method: GET
URL: /api/Guide/GetAllMedia/:guideId
Authentication: No
Response Example:
{ “success”: true, “data”: [ { “id”: 12, “created_at”: “2025-04-01T10:00:00Z”, “media_data”: “data:image/jpeg;base64,…” } ] }

⸻

	9.	Delete Guide Media
Method: DELETE
URL: /api/Guide/DeleteMedia/:mediaId
Authentication: Yes (JWT)
Response Example:
{ “success”: true, “message”: “Media deleted successfully” }

⸻

Booking APIs:
	10.	Make Booking Request
Method: POST
URL: /api/Bookings/Request
Authentication: No
Body Example:
{ “guide_id”: 23, “traveler_id”: 5, “date”: “27.04.2025” }
Response Example:
{ “message”: “Booking request created”, “booking_id”: 120 }

⸻

	11.	Guide Accepts Booking
Method: POST
URL: /api/Bookings/Accept
Authentication: No
Body Example:
{ “booking_id”: 120 }

⸻

	12.	Guide Declines Booking
Method: POST
URL: /api/Bookings/Decline
Authentication: No
Body Example:
{ “booking_id”: 120 }

⸻

	13.	Guide Cancels Booking
Method: POST
URL: /api/Bookings/Cancel
Authentication: No
Body Example:
{ “booking_id”: 120 }

⸻

	14.	Traveler Cancels Booking
Method: POST
URL: /api/Bookings/Traveler/CancelBooking
Authentication: No
Body Example:
{ “booking_id”: 120, “traveler_id”: 5 }

⸻

	15.	Traveler Leaves Review
Method: POST
URL: /api/Bookings/Traveler/LeaveReview
Authentication: No
Body Example:
{ “booking_id”: 120, “traveler_id”: 5, “rate”: 5, “review”: “Great experience!” }

⸻

	16.	Get Guide Bookings
Method: GET
URL: /api/GuideBookings
Authentication: No
Query Example:
?guide_id=23&start=27.04.2025&end=31.12.2025&confirmed=true&pending=true&completed=true

⸻

	17.	Get Traveler Bookings
Method: GET
URL: /api/TravelerBookings
Authentication: No
Query Example:
?traveler_id=5&start=27.04.2025&end=31.12.2025&confirmed=true&pending=true&completed=true
